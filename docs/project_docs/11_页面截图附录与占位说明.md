# 页面截图附录与占位说明

## 1. 编写目的

本文档用于指导论文与答辩中的页面截图选取、图题撰写和配图说明写法，使截图不仅体现界面外观，还能准确传达系统功能与工程价值。

## 2. 截图使用原则

1. 优先选择能够体现任务化管理与统一组织能力的页面。
2. 每张截图都应说明“页面展示了什么”以及“页面体现了什么系统能力”。
3. 图题尽量采用正式、简洁、可直接放入论文的写法。

## 3. 页面截图清单

### 3.1 系统总览页

建议展示内容：

1. 统计卡片。
2. 服务状态区域。
3. 最近任务列表。

建议图题：

“图 X-X 系统总览页面”

建议说明文字：

“系统总览页集中展示数据集数量、任务数量、模型版本数量以及服务状态信息，是说明系统整体运行结构与管理能力的核心页面。”

### 3.2 数据集管理页

建议展示内容：

1. 数据集列表。
2. 上传入口。
3. 数据集元信息面板。

建议图题：

“图 X-X 数据集管理页面”

建议说明文字：

“数据集管理页用于统一维护标准数据包，支持后续推理任务和训练任务调用，体现了系统对多模型输入数据的统一组织能力。”

### 3.3 推理任务页

建议展示内容：

1. 模型选择区域。
2. 数据集选择区域。
3. 参数输入区域。
4. 任务状态或结果摘要区域。

建议图题：

“图 X-X 推理任务页面”

建议说明文字：

“推理任务页支持用户以任务化方式提交预测请求，并查看任务状态与结果摘要，体现了系统对四模型推理流程的统一封装。”

### 3.4 训练任务页

建议展示内容：

1. 模型选择区域。
2. 训练参数区域。
3. 任务列表或任务详情。
4. 版本关联信息。

建议图题：

“图 X-X 训练任务页面”

建议说明文字：

“训练任务页用于组织模型训练请求，并将训练任务与训练版本目录建立关联，体现了系统对训练过程的规范化管理能力。”

### 3.5 模型版本页

建议展示内容：

1. 版本列表。
2. 模型文件、指标文件和说明文件摘要。
3. 来源任务信息。

建议图题：

“图 X-X 模型版本管理页面”

建议说明文字：

“模型版本页用于展示训练结果的归档情况，支持查看版本指标、参数与来源任务信息，体现了系统结果可追溯性设计。”

### 3.6 系统说明页

建议展示内容：

1. 系统定位说明。
2. 四模型统一关系说明。
3. 功能导航入口。

建议图题：

“图 X-X 系统说明页面”

建议说明文字：

“系统说明页用于概括系统定位、总体能力和模块关系，适合作为第 4 章系统实现总述的配图。”

### 3.7 模型说明页

建议展示内容：

1. 四模型名称。
2. 模型输入差异摘要。
3. 统一接入方式说明。

建议图题：

“图 X-X 模型说明页面”

建议说明文字：

“模型说明页用于展示四模型的基本定位与输入差异，并说明系统如何通过统一数据包和统一任务接口对模型进行组织。”

## 4. 论文配图写法模板

可直接套用以下模板：

“图 X-X 展示了____页面。该页面主要用于____，能够体现系统在____方面的设计与实现。”

示例：

“图 X-X 展示了推理任务页面。该页面主要用于提交药物组合协同预测任务并查看任务状态，能够体现系统在统一推理流程组织方面的设计与实现。”

## 5. 答辩展示顺序建议

1. 先展示系统总览页。
2. 再展示数据集管理页。
3. 然后展示推理任务页。
4. 再展示训练任务页。
5. 最后展示模型版本页。

## 6. 占位说明写法

当真实截图尚未补充时，可先使用以下占位文字：

1. `[截图占位：系统总览页面]`
2. `[截图占位：数据集管理页面]`
3. `[截图占位：推理任务页面]`
4. `[截图占位：训练任务页面]`
5. `[截图占位：模型版本页面]`
