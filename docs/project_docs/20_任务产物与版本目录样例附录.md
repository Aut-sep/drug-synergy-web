# 任务产物与版本目录样例附录

## 1. 编写目的

本文档用于展示本系统中推理任务产物、训练任务产物和训练版本目录的典型组织方式，便于论文第 4 章和第 5 章直接引用。

## 2. 推理输出目录样例

```text
<推理输出目录>/
  infer_20260428_001/
    request.json
    result_summary.json
    prediction.csv
    run.log
```

说明：

1. `request.json` 保存任务输入摘要。
2. `result_summary.json` 保存结果摘要信息。
3. `prediction.csv` 保存预测结果明细。
4. `run.log` 保存运行日志。

## 3. 训练输入包目录样例

```text
<训练输入包目录>/
  train_20260428_001/
    dataset_meta.json
    train_samples.csv
    valid_samples.csv
    config.json
```

说明：

1. 训练输入包用于保存一次训练任务所需的标准化输入。
2. `config.json` 用于保存训练参数摘要。

## 4. 训练版本目录样例

```text
<训练版本目录>/
  dualsyn/
    version_20260428_001/
      model.bin
      metrics.json
      params.json
      summary.md
      train.log
```

说明：

1. 版本目录以模型名称和版本编号组织。
2. 版本目录中至少包含模型文件、指标文件、参数文件和日志文件。

## 5. 目录设计价值

1. 便于按任务追溯输入与输出。
2. 便于按版本追溯训练结果。
3. 便于在答辩中说明系统的工程化归档能力。

## 6. 对论文的直接用途

适合放在：

1. 第 4 章的数据与目录设计小节。
2. 第 5 章的结果追溯性说明小节。

## 7. 可直接引用的总结语

“系统通过推理输出目录和训练版本目录对实验产物进行分层归档，使一次任务的输入、过程和输出能够形成完整的可追溯链路。”
