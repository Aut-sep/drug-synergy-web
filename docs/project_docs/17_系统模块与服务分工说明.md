# 系统模块与服务分工说明

## 1. 目的

本文档用于明确当前原生 Web 系统中各模块、各服务和各目录的职责边界，便于论文编写、答辩讲解和后续维护。

## 2. 总体分工

| 层次 | 对象 | 主要职责 |
| --- | --- | --- |
| 表现层 | `web_system_frontend/` | 页面展示、表单交互、结果预览、轮询刷新 |
| 业务层 | `web_system_backend/` | 接口路由、参数校验、任务登记、状态汇总、下载封装 |
| 运行时层 | `web_system_runtime/` | 推理网关、模型服务包装、训练服务入口、运行时存储 |
| 模型层 | 模型仓库与 `benchmark_factory/` | 承载具体模型实现与脚本 |
| 存储层 | SQLite 与运行时目录 | 持久化元数据、上传包、结果、版本资产 |

## 3. 前端分工

前端模块承担以下职责：

1. 路由组织：
   仪表盘、系统说明、数据集、推理、训练、版本、任务详情等页面。
2. 表单交互：
   数据上传、推理参数选择、训练参数填写。
3. 状态刷新：
   轮询推理任务和训练任务状态。
4. 结果展示：
   显示结果表格、日志、版本信息和下载入口。

前端不负责：

1. 直接访问模型服务。
2. 直接拼接本地磁盘路径。
3. 直接操作数据库。

## 4. 后端分工

后端模块承担以下职责：

1. 注册 `system`、`datasets`、`inference`、`training`、`versions` 路由。
2. 管理 SQLite 中的数据集和任务元数据。
3. 校验数据集是否满足推理或训练要求。
4. 向推理网关和训练服务发起标准化请求。
5. 将远端状态统一映射为本地状态。
6. 生成推理结果下载和训练产物下载入口。
7. 汇总系统概览与健康状态。

### 4.1 路由层责任

| 路由模块 | 责任 |
| --- | --- |
| `system.py` | 概览统计、服务健康摘要、资源快照 |
| `datasets.py` | 数据集列表、详情、预览、上传 |
| `inference.py` | 推理任务列表、创建、刷新、详情、取消、结果下载 |
| `training.py` | 训练任务列表、创建、刷新、详情、取消、产物下载 |
| `versions.py` | 模型版本列表 |

### 4.2 服务层责任

| 服务模块 | 责任 |
| --- | --- |
| `legacy_support.py` | 运行时桥接、远端服务调用、数据包保存 |
| `serializers.py` | 数据对象序列化、状态标准化、路径解析 |
| `task_details.py` | 任务详情聚合、结果预览、训练产物文件列表构造 |

## 5. 运行时层分工

运行时层承担“实际执行”职责，具体包括：

### 5.1 推理网关

推理网关负责：

1. 接收后端提交的标准化推理请求。
2. 向不同模型服务分发推理任务。
3. 统一返回任务状态、日志和输出位置。

### 5.2 模型服务包装层

四个模型服务分别负责各自模型的推理执行：

1. `dualsyn-service`
2. `mfsyndcp-service`
3. `mvcasyn-service`
4. `mtlsynergy-service`

这些服务均由 `web_system_runtime/docker-compose.yml` 管理。

### 5.3 训练服务

训练服务负责：

1. 接收训练任务请求。
2. 调用模型训练脚本。
3. 将产物写入版本目录。
4. 暴露训练任务和模型版本查询接口。

当前训练服务通过 WSL 启动，不包含在 `web_system_runtime/docker-compose.yml` 中。

## 6. 存储目录分工

### 6.1 后端持久化

| 路径 | 用途 |
| --- | --- |
| `web_system_backend/data/web_system.db` | 数据集和任务元数据 |
| `web_system_backend/outputs/downloads/` | 后端生成的压缩下载文件 |

### 6.2 运行时存储

| 路径 | 用途 |
| --- | --- |
| `web_system_runtime/user_bundles/web_system_uploads/` | 用户上传数据包 |
| `web_system_runtime/outputs/web_system_bridge/` | 推理输出 |
| `web_system_runtime/trained_model_versions/` | 训练版本产物 |
| `web_system_runtime/training_bundles/` | 内置或整理后的训练数据包 |

## 7. 脚本分工

| 脚本 | 责任 |
| --- | --- |
| `scripts/start_web_system.ps1` | 本地启动前端、后端，并按需启动推理栈和训练服务 |
| `scripts/start_web_system_docker.ps1` | 容器化启动 Web 前后端 |
| `scripts/check_web_system.ps1` | 健康检查与演示前巡检 |

## 8. 分工结论

当前系统已经形成清晰的职责划分：

1. 前端只负责用户交互与呈现。
2. 后端只负责业务编排与资源控制。
3. 运行时层只负责推理和训练执行。
