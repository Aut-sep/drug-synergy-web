## 4. 代码与运行对象的中性称谓

为保证论文、答辩和系统文档口径一致，本文档统一采用以下中性称谓：

- 管理前端：负责页面展示、任务提交和结果查看的界面层。
- 业务后端：负责接口编排、任务登记、状态汇总和文件组织的服务层。
- 模型执行组件：负责四个药物协同预测模型的实际运行。
- 推理服务组件：负责推理请求接收、参数组织与结果回传。
- 训练服务组件：负责训练任务接收、训练过程调度与版本产物沉淀。
- 系统工作目录：系统运行时所使用的数据、任务、日志和结果目录集合。
- 推理输出目录：保存推理结果文件、可视化文件和任务日志的目录。
- 训练版本目录：保存训练版本、模型文件、指标文件和说明文件的目录。
