# 部署与运行手册

## 1. 运行组成

当前系统由三部分组成：

1. Web 前端：`web_system_frontend/`
2. Web 后端：`web_system_backend/`
3. 运行时层：`web_system_runtime/`

其中运行时层又分为两类能力：

1. 推理运行时：网关和四个模型服务，使用 Docker Compose 管理。
2. 训练服务：当前通过 WSL 单独启动。

## 2. 推荐启动顺序

### 2.1 首次构建推理运行时

首次启动或镜像失效时，先在项目根目录执行：

```powershell
docker compose -f .\web_system_runtime\docker-compose.yml up -d --build
```

该命令会构建并启动：

1. `gateway`
2. `dualsyn-service`
3. `mfsyndcp-service`
4. `mvcasyn-service`
5. `mtlsynergy-service`

### 2.2 本地启动 Web 系统

如需以前后端本地进程方式启动系统，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system.ps1
```

该脚本会：

1. 检查 Docker Desktop 是否可用。
2. 复用已构建好的推理运行时镜像并启动推理栈。
3. 启动后端 `uvicorn app.main:app --port 9000`。
4. 启动前端 `npm run dev -- --host 127.0.0.1 --port 5173`。

### 2.3 启动训练服务

如需同时启用训练服务，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system.ps1 -StartTrainingService
```

该脚本会在 WSL 中启动：

```text
service_runtime.training_service:app
```

默认监听端口：

```text
8011
```

### 2.4 容器化启动 Web 层

如需将前端和后端作为容器启动，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system_docker.ps1
```

如镜像已构建，可跳过构建：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system_docker.ps1 -NoBuild
```

| --- | --- |
| 前端 | `http://127.0.0.1:5173` |

| 训练服务健康检查 | `http://127.0.0.1:8011/health` |

## 4. 环境准备

### 4.1 必需环境

推荐准备以下基础环境：

1. Python
2. Node.js / npm
3. Docker Desktop
4. WSL

### 4.2 前端依赖安装

首次运行前端时执行：

```powershell
cd D:\codex\bishe_base\web_system_frontend
npm install
```

### 4.3 后端依赖安装

首次运行后端时执行：

```powershell
cd D:\codex\bishe_base\web_system_backend
python -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

## 5. 配置项

后端核心配置项如下：

| 变量名 | 默认含义 |
| --- | --- |
| `WEB_SYSTEM_DATABASE_URL` | SQLite 数据库连接串 |
| `WEB_SYSTEM_RUNTIME_ROOT` | 运行时目录根路径 |
| `WEB_SYSTEM_WORKSPACE_ROOT` | 工作区根路径 |
| `WEB_SYSTEM_GATEWAY_URL` | 推理网关地址 |
| `WEB_SYSTEM_TRAINING_URL` | 训练服务地址 |
| `WEB_SYSTEM_GATEWAY_RUNTIME_ROOT` | 网关容器中的运行时根挂载路径 |

## 6. 健康检查

执行如下命令可对主要入口进行巡检：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\check_web_system.ps1
```

该脚本检查以下地址：

1. 前端首页
2. 后端 `/health`
3. 后端 `/api/system/summary`
4. 推理网关 `/health`
5. 训练服务 `/health`

其中训练服务为非强制项，其余四项失败时脚本会返回非零退出码。

## 7. 常见运行场景

### 7.1 本地演示场景

推荐命令：

```powershell
docker compose -f .\web_system_runtime\docker-compose.yml up -d --build
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system.ps1
```

如需训练演示：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system.ps1 -StartTrainingService
```

### 7.2 仅启动前后端容器

推荐命令：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system_docker.ps1
```

若推理栈未运行，还需单独启动：

```powershell
docker compose -f .\web_system_runtime\docker-compose.yml up -d --build
```

### 7.3 推理栈已常驻运行

若 `8000` 端口已监听，可执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_web_system.ps1 -SkipDocker
```

## 8. 日志与输出位置

本地脚本启动时的主要日志位置如下：

| 位置 | 说明 |
| --- | --- |
| `web_system_backend/runtime_logs/backend.out.log` | 后端标准输出 |
| `web_system_backend/runtime_logs/backend.err.log` | 后端错误输出 |
| `web_system_frontend/runtime_logs/frontend.out.log` | 前端标准输出 |
| `web_system_frontend/runtime_logs/frontend.err.log` | 前端错误输出 |

运行时输出位置如下：

| 位置 | 说明 |
| --- | --- |
| `web_system_runtime/user_bundles/web_system_uploads/` | 上传数据包 |
| `web_system_runtime/outputs/web_system_bridge/` | 推理输出 |
| `web_system_runtime/trained_model_versions/` | 训练版本目录 |
| `web_system_backend/outputs/downloads/` | 后端打包下载文件 |

## 9. 故障排查建议

1. 页面打不开时，先检查 `5173` 端口和前端日志。
2. 首页摘要加载失败时，先检查 `9000/health`。
3. 无法提交推理任务时，检查 `8000/health` 和推理栈容器状态。
4. 无法提交训练任务时，检查 `8011/health` 和 WSL 训练服务进程。
5. 结果无法下载时，检查后端返回的 `output_path_host` 是否存在对应文件。

## 10. 现役部署结论

当前运行手册仅围绕以下对象展开：

1. `docker-compose.web.yml`
2. `web_system_runtime/docker-compose.yml`
3. `scripts/start_web_system.ps1`
4. `scripts/start_web_system_docker.ps1`
