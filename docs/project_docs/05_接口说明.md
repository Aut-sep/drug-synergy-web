# 接口说明

## 1. 说明范围

本文档描述当前原生 Web 系统后端对外提供的实际接口，接口来源为 `web_system_backend/app/routers/`。

## 2. 接口设计原则

1. 接口按业务资源分组。
2. 前端统一通过 `/api` 调用后端。
3. 推理与训练任务均以本地任务记录为主线。
4. 后端负责将远端状态统一映射为本地状态。
5. 文件下载必须经由后端校验后输出。

## 3. 通用接口

### 3.1 健康检查

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/health` | 检查后端服务本身是否启动 |

返回示例：

```json
{
  "status": "ok",
  "service": "web_system_backend"
}
```

### 3.2 系统概览

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/system/summary` | 获取首页概览、服务健康状态和资源快照 |

核心返回字段：

1. `dataset_count`
2. `inference_run_count`
3. `training_run_count`
4. `running_run_count`
5. `latest_model_version_count`
6. `gateway_health`
7. `training_health`
8. `resource_snapshot`

## 4. 数据集接口

### 4.1 接口列表

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/datasets` | 获取数据集列表 |
| `GET` | `/api/datasets/{dataset_id}` | 获取单个数据集详情 |
| `GET` | `/api/datasets/{dataset_id}/preview` | 预览样本文件前若干行 |
| `POST` | `/api/datasets/upload` | 上传数据集文件包 |

### 4.2 数据集对象

`DatasetBundleRead` 关键字段如下：

| 字段 | 含义 |
| --- | --- |
| `id` | 数据集编号 |
| `name` | 数据集名称 |
| `bundle_path` | 数据集物理目录 |
| `sample_file` | 样本文件名 |
| `source_type` | 来源类型 |
| `description` | 描述 |
| `is_ready` | 是否通过推理样本校验 |
| `sample_count` | 样本数量 |
| `files` | 文件列表 |
| `validation_detail` | 校验说明 |
| `bundle_kind` | `inference`、`training`、`hybrid`、`invalid` |
| `supports_inference` | 是否支持推理 |
| `supports_training` | 是否支持训练 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

### 4.3 数据预览对象

`DatasetPreviewRead` 关键字段如下：

1. `dataset_id`
2. `sample_file`
3. `columns`
4. `preview_rows`
5. `preview_row_count`
6. `note`

### 4.4 上传接口说明

请求类型：

`multipart/form-data`

表单字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | string | 是 | 数据集名称 |
| `description` | string | 否 | 数据集说明 |
| `files` | file[] | 是 | 一个或多个 CSV 文件 |

接口行为：

1. 自动清理文件名。
2. 将文件保存到 `web_system_runtime/user_bundles/web_system_uploads/`。
3. 自动校验样本结构。
4. 写入 SQLite 中的数据集记录。

## 5. 推理任务接口

### 5.1 接口列表

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/inference-runs` | 获取推理任务列表 |
| `POST` | `/api/inference-runs` | 创建推理任务 |
| `GET` | `/api/inference-runs/{task_id}` | 获取推理任务当前状态 |
| `GET` | `/api/inference-runs/{task_id}/detail` | 获取推理任务详情 |
| `POST` | `/api/inference-runs/{task_id}/cancel` | 取消推理任务 |
| `GET` | `/api/inference-runs/{task_id}/download` | 下载推理结果 CSV |

### 5.2 创建请求体

`InferenceRunCreate` 字段如下：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `dataset_id` | integer | 推理数据集编号 |
| `selected_models` | string[] | 参与推理的模型集合，至少一个 |
| `model_version_id` | string | 模型版本编号，默认 `__default__` |

请求示例：

```json
{
  "dataset_id": 3,
  "selected_models": ["DualSyn", "MTLSynergy"],
  "model_version_id": "__default__"
}
```

### 5.3 任务对象

推理任务返回 `RunTaskRead`，关键字段如下：

| 字段 | 含义 |
| --- | --- |
| `id` | 本地任务编号 |
| `task_type` | 固定为 `inference` |
| `title` | 任务标题 |
| `local_status` | 本地标准化状态 |
| `remote_state` | 远端原始状态 |
| `remote_run_id` | 远端任务编号 |
| `dataset_id` | 所属数据集编号 |
| `model_version_id` | 使用的模型版本 |
| `selected_models` | 模型集合 |
| `output_path` | 远端返回的输出路径 |
| `output_path_host` | 后端解析后的宿主机路径 |
| `output_available` | 本地输出是否存在 |
| `result_download_endpoint` | 推理结果下载接口 |
| `log_excerpt` | 日志摘要 |
| `error_message` | 错误信息 |
| `summary` | 远端返回的摘要信息 |
| `detail_endpoint` | 详情接口 |

### 5.4 详情对象补充字段

推理详情返回 `RunTaskDetailRead`，除 `RunTaskRead` 外还包含：

1. `dataset`
2. `log_text`
3. `log_full_available`
4. `result_preview_columns`
5. `result_preview_rows`
6. `service_outputs`
7. `resource_reports`

### 5.5 推理任务状态说明

后端会在查询列表或单条详情时主动刷新远端任务状态。当前统一状态集合为：

1. `waiting`
2. `running`
3. `completed`
4. `failed`
5. `canceling`
6. `canceled`

## 6. 训练任务接口

### 6.1 接口列表

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/training-runs` | 获取训练任务列表 |
| `POST` | `/api/training-runs` | 创建训练任务 |
| `GET` | `/api/training-runs/{task_id}` | 获取训练任务当前状态 |
| `GET` | `/api/training-runs/{task_id}/detail` | 获取训练任务详情 |
| `POST` | `/api/training-runs/{task_id}/cancel` | 取消训练任务 |
| `GET` | `/api/training-runs/{task_id}/artifacts/archive` | 打包下载训练产物目录 |
| `GET` | `/api/training-runs/{task_id}/artifacts/{artifact_path:path}` | 下载单个训练产物文件 |

### 6.2 创建请求体

`TrainingRunCreate` 字段如下：

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `dataset_id` | integer | 无 | 训练数据集编号 |
| `selected_models` | string[] | 无 | 训练模型集合，至少一个 |
| `profile` | string | `quick` | 训练档位 |
| `device` | string | `auto` | 运行设备 |
| `epochs` | integer \| null | `null` | 训练轮数，可留空 |
| `label_threshold` | number | `10.0` | 阈值标签参数 |
| `version_note` | string | `""` | 版本备注 |

请求示例：

```json
{
  "dataset_id": 8,
  "selected_models": ["DualSyn", "MVCASyn"],
  "profile": "quick",
  "device": "auto",
  "epochs": 5,
  "label_threshold": 10.0,
  "version_note": "smoke run"
}
```

### 6.3 详情对象补充字段

训练详情同样返回 `RunTaskDetailRead`，其中额外重点字段为：

1. `artifact_files`
2. `artifact_file_items`
3. `manifest`
4. `service_outputs`
5. `resource_reports`

`artifact_file_items` 中每一项包含：

1. `path`
2. `size_bytes`
3. `download_endpoint`

### 6.4 训练数据要求

训练任务创建前，后端会检查训练数据包中是否至少包含以下文件：

1. `samples.csv`
2. `drugs.csv`
3. `cells_dualsyn.csv`
4. `cells_mtl.csv`
5. `cells_mvc_exp.csv`
6. `cells_mvc_cn.csv`

## 8. 错误处理

### 8.1 常见状态码

| 状态码 | 含义 |
| --- | --- |
| `200` | 请求成功 |
| `400` | 参数错误、文件不合法、路径非法 |
| `404` | 数据集、任务或产物不存在 |
| `502` | 后端桥接远端推理/训练服务失败 |

### 8.2 错误设计原则

1. 本地校验错误返回 `400`。
2. 数据不存在返回 `404`。
3. 推理网关或训练服务不可用时返回 `502`。
4. 对训练产物下载路径执行边界校验，防止越界访问。
