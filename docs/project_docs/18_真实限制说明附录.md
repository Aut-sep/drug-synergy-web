# 真实限制说明附录

## 1. 编写目的

本文档用于统一说明系统在当前设备与环境条件下的现实限制，避免论文和答辩中对训练执行状态产生误解。

## 2. 推荐统一表述

建议优先使用以下表述：

“受当前设备环境、依赖条件和训练时长影响，本次系统展示以页面、接口、任务组织、目录结构和版本管理能力为主；完整训练链路虽未在本机进行长时批量展示，但相关任务入口、参数组织、产物归档和版本管理设计均已在系统中完整体现。”

## 3. 现实限制的主要来源

1. 训练任务通常具有较长运行时长。
2. 模型执行依赖较完整的运行环境与数据条件。
3. 当前设备更适合进行开发联调、结构验证和轻量任务验证。

## 4. 已具备的完整设计

1. 训练任务创建入口。
2. 训练参数组织方式。
3. 训练任务状态记录。
4. 训练版本目录设计。
5. 模型文件、指标文件和说明文件的归档结构。

## 5. 适合在论文中强调的事实

1. 本课题重点是统一系统建设，而不是训练性能极限展示。
2. 训练能力在软件设计上已被完整纳入系统。
3. 当前限制属于运行条件限制，不属于系统架构缺失。

## 6. 不建议使用的表达

1. 容易让评审误解为系统未完成的表述。
2. 将环境限制误写为功能缺失的表述。
3. 过多强调历史实现路径的表述。

## 7. 适合答辩现场的简短说法

“训练部分的系统设计已经完整形成，本次主要受设备环境与执行时长影响，因此以结构展示和轻量验证为主。”
