# 四模型输入差异对照表

## 1. 编写目的

本文档用于说明 `DualSyn`、`MFSynDCP`、`MVCASyn`、`MTLSynergy` 四个模型在输入结构上的主要差异，以及本系统如何通过统一数据包进行组织。

## 2. 差异总表

| 模型 | 药物信息 | 细胞系信息 | 标签信息 | 额外说明 |
| --- | --- | --- | --- | --- |
| `DualSyn` | 需要药物特征或药物表示 | 需要细胞系特征 | 需要协同标签 | 侧重双药与细胞联合输入 |
| `MFSynDCP` | 需要药物分子或特征表示 | 需要细胞系表达信息 | 需要协同标签 | 数据组织强调矩阵化输入 |
| `MVCASyn` | 需要药物特征与组合关系 | 需要细胞系特征 | 需要协同标签 | 可能包含多视角输入组织 |
| `MTLSynergy` | 需要药物组合输入 | 需要细胞系特征 | 需要协同标签及相关任务标签 | 适合多任务场景描述 |

## 3. 统一数据包映射关系

| 统一数据包字段 | 对应含义 | 四模型共用情况 |
| --- | --- | --- |
| 药物对象 | 保存药物编号、名称与特征数据 | 共用 |
| 细胞系对象 | 保存细胞系编号与特征数据 | 共用 |
| 样本配对对象 | 保存药物 A、药物 B 与细胞系关系 | 共用 |
| 协同标签对象 | 保存协同分值或分类标签 | 共用 |
| 模型扩展字段 | 保存模型特定补充输入 | 按需使用 |

## 4. 系统整合思路

1. 先抽取四模型共享字段。
2. 再为模型特定字段预留扩展区域。
3. 在推理服务组件与训练服务组件中完成输入适配。
4. 对外保持统一任务接口，对内完成模型差异处理。

## 5. 对论文第 4 章的直接用途

本表最适合放在以下位置：

1. 统一数据包设计小节。
2. 四模型整合实现小节。
3. 模型说明页配套表格。

## 6. 可直接引用的总结语

“四模型在药物特征组织、细胞系特征组织和任务标签组织上存在差异，因此系统采用统一数据包加模型扩展字段的方式完成输入抽象，在对外保持统一接口的同时保留模型内部所需差异信息。”
