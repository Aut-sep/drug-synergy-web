# 系统演示脚本与答辩演示流程

## 1. 演示目标

通过有限时间内的页面展示和流程说明，让评审快速理解本系统的定位、结构、能力与现实限制。

## 2. 推荐演示顺序

1. 课题背景与系统目标。
2. 系统总览页。
3. 数据集管理页。
4. 推理任务页。
5. 训练任务页。
6. 模型版本页。
7. 限制说明与总结。

## 3. 五分钟精简版脚本

### 3.1 开场

“本课题聚焦于四个已有药物组合协同预测模型的软件工程整合，目标是构建一套统一的数据、任务、结果和版本管理系统，而不是提出新的预测模型。”

### 3.2 展示系统总览页

“这里展示的是系统总览页，可以看到数据集、任务和模型版本的整体统计信息，也可以看到系统服务状态和最近任务情况。”

### 3.3 展示推理任务页

“在推理任务页中，用户可以选择模型和数据集，提交预测任务，并在任务完成后查看结果摘要与输出信息。”

### 3.4 展示训练任务页

“在训练任务页中，系统提供了统一的训练任务入口，能够记录训练参数、训练状态以及与训练版本目录的关联关系。”

### 3.5 展示模型版本页

“模型版本页用于集中管理训练产物，支持查看模型文件、指标文件和来源任务信息，体现系统对实验结果的归档与追溯能力。”

### 3.6 结束语

“整体来看，本系统已经完成了四模型统一系统化组织的主要工程工作，训练长时执行受当前环境影响，但系统设计与文档交付已经完整形成。”

## 4. 十分钟完整版建议

1. 用 1 分钟说明课题定位。
2. 用 2 分钟讲总体架构。
3. 用 2 分钟讲数据集与推理流程。
4. 用 2 分钟讲训练任务与版本目录。
5. 用 1 分钟讲测试与限制。
6. 用 2 分钟回答问题。

## 5. 现场兜底表述

### 5.1 训练未做完整长时展示时

“当前设备环境主要用于系统实现与轻量验证，因此训练长时执行不作为演示主线；但训练任务入口、版本目录和产物组织已经在系统中完整体现。”

### 5.2 被问到论文重点时

“论文重点是统一系统的设计与实现，强调的是工程整合与组织能力，而不是新的模型结构创新。”

## 6. PPT 页与演示动作对应关系

| PPT 页 | 演示动作 |
| --- | --- |
| 课题背景 | 概述系统目标 |
| 总体架构 | 指向架构图解释分层 |
| 系统总览 | 打开总览页说明系统全貌 |
| 推理流程 | 切到推理任务页演示任务提交逻辑 |
| 训练流程 | 切到训练任务页说明训练组织方式 |
| 版本管理 | 打开模型版本页说明结果归档 |
| 测试与限制 | 用测试记录和限制说明收尾 |
