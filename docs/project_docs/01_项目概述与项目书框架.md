# 项目概述与项目书框架

## 1. 项目名称

基于深度学习的药物组合协同预测系统设计与实现

## 2. 项目背景

药物组合协同预测是智能药物发现中的重要问题。现有研究已经提出了多种深度学习模型，但不同模型在输入格式、特征组织、训练方式和结果表达上存在明显差异，导致模型难以在统一软件平台中进行管理和展示。

本课题的核心工作并非提出新的预测模型，而是围绕 `DualSyn`、`MFSynDCP`、`MVCASyn`、`MTLSynergy` 四个已有模型，建设一个统一的软件系统。系统通过统一数据包、统一任务编排、统一结果展示和统一版本管理，将模型研究成果转化为可运行、可演示、可扩展的软件工程交付物。

## 3. 建设目标

### 3.1 总体目标

建设一套面向药物组合协同预测场景的综合系统，使用户能够在统一界面下完成数据集管理、推理任务提交、训练任务组织、模型版本查看、结果回溯和系统状态观测。

### 3.2 具体目标

1. 建立统一数据包规范，降低四模型输入差异带来的使用门槛。
2. 建立统一推理流程，使用户能够按任务方式提交预测请求并查看结果。
3. 建立统一训练流程，使训练任务、训练产物和训练版本能够纳入一致的管理框架。
4. 建立统一结果展示机制，使系统能够围绕任务、版本和指标组织实验输出。
5. 建立工程化文档体系，为毕业论文、答辩材料和后续维护提供依据。

## 4. 与论文主线的对齐说明

本课题论文主线应表述为“多模型统一预测软件系统的设计与实现”，而不是“新的药物协同预测模型设计”。论文中应突出以下几点：

1. 研究对象是四个已有深度学习模型的软件工程整合。
2. 创新点主要体现在统一数据组织、统一任务编排、统一结果管理和统一界面呈现。
3. 工程价值体现在将分散的模型能力整理为可部署、可演示、可复用的系统。
4. 在当前设备条件下，训练长时执行能力受到环境限制，但系统设计、接口组织和版本管理方案完整。

## 5. 项目范围

### 5.1 已实现范围

1. 管理前端页面体系。
2. 业务后端接口体系。
3. 系统总览、数据集管理、推理任务管理、训练任务管理、模型版本管理等页面。
4. 统一数据包校验与任务输入组织能力。
5. 推理结果文件、任务记录和版本产物的目录化管理。
6. 面向论文与答辩的工程文档体系。

### 5.2 设计已覆盖但执行受环境限制的范围

1. 四模型完整训练任务的长时批量执行。
2. 更大规模数据集条件下的持续训练验证。
3. 多用户并发环境下的长期稳定性验证。

## 6. 目标用户

1. 毕业设计作者：用于开展实验组织、系统实现和论文撰写。
2. 指导教师与评审教师：用于查看系统能力、演示流程和结果组织方式。
3. 后续研究人员：用于在统一框架下继续扩展模型和实验。

## 7. 系统交付物

1. 可运行的管理前端。
2. 可调用的业务后端。
3. 统一数据包规范与目录规范。
4. 推理任务与训练任务管理机制。
5. 训练版本目录与推理输出目录。
6. 覆盖需求、设计、接口、数据、部署、测试和答辩支撑的完整文档集。

## 8. 实施条件与现实约束

1. 当前设备具备基础开发与调试能力，可完成页面、接口、任务组织和轻量验证。
2. 训练执行依赖模型环境、算力条件和数据准备情况，因此完整训练展示受现实环境约束。
3. 即使训练长时执行受限，系统在软件设计上仍已预留训练任务入口、参数组织、版本记录和结果归档能力。

## 10. 可直接引用的总结表述

可在论文中使用如下概括性表述：

“本课题以四个已有药物组合协同预测模型为基础，围绕统一数据组织、统一任务管理、统一结果展示和统一版本沉淀开展系统建设，目标是形成一套可运行、可演示、可扩展的药物组合协同预测软件系统，为模型应用落地和毕业设计答辩提供完整工程支撑。”
