# 变更记录与迭代记录

## 1. 目的

本文档用于梳理系统从课题原型到工程化交付材料的演进过程，突出系统建设重点和文档完善过程。

## 2. 迭代阶段划分

| 阶段 | 名称 | 主要内容 |
| --- | --- | --- |
| 阶段 1 | 课题原型整理阶段 | 明确四模型、数据来源与实验目标 |
| 阶段 2 | 统一数据组织阶段 | 形成统一数据包和目录组织思路 |
| 阶段 3 | 任务化管理阶段 | 形成推理任务、训练任务和版本管理框架 |
| 阶段 4 | 页面与接口完善阶段 | 完成页面体系、接口体系与状态展示 |
| 阶段 5 | 文档交付阶段 | 完成需求、设计、部署、测试和答辩文档 |

## 3. 关键里程碑

1. 明确论文主线为四模型统一系统设计与实现。
2. 建立统一数据包组织方案。
3. 建立推理任务与训练任务的统一记录方式。
4. 建立训练版本目录与推理输出目录。
5. 建立可直接用于论文与答辩的文档体系。

## 5. 后续可继续补充的记录项

1. 页面截图替换进度。
