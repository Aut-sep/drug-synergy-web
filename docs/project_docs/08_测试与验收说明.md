# 测试与验收说明

## 1. 文档目的

本文档用于说明本系统的测试范围、测试方法、验收标准和现实限制，为论文第 5 章和项目交付验收提供依据。

## 2. 测试目标

1. 验证主要页面可访问。
2. 验证主要接口可访问。
3. 验证推理任务链路能够完成基本流转。
4. 验证训练任务入口、版本登记和目录组织逻辑完整。
5. 验证系统文档与系统实现口径一致。

## 3. 测试范围

### 3.1 页面测试

1. 系统总览页。
2. 数据集管理页。
3. 推理任务页。
4. 训练任务页。
5. 模型版本页。

### 3.2 接口测试

1. 健康检查接口。
2. 系统总览接口。
3. 数据集接口。
4. 推理任务接口。
5. 训练任务接口。
6. 模型版本接口。

### 3.3 目录与产物测试

1. 上传数据包目录。
2. 推理输出目录。
3. 训练版本目录。
4. 日志目录。

## 4. 测试方法

1. 页面访问测试。
2. 接口请求测试。
3. 任务创建与查询测试。
4. 目录生成与文件留痕检查。
5. 文档与实现一致性检查。

## 5. 典型测试项

| 编号 | 测试内容 | 预期结果 |
| --- | --- | --- |
| T01 | 访问 `/health` | 返回正常状态 |
| T02 | 访问 `/api/system/summary` | 返回系统概览、任务统计与服务状态 |
| T03 | 打开系统总览页 | 页面正常显示统计信息 |
| T04 | 打开数据集管理页 | 页面正常显示数据集入口 |
| T05 | 提交推理任务 | 返回任务编号并生成任务记录 |
| T06 | 查询推理任务详情 | 能查看状态、结果摘要和输出目录 |
| T07 | 创建训练任务 | 返回任务编号并生成训练记录 |
| T08 | 查看模型版本页 | 能查看版本列表与版本摘要 |
| T09 | 检查输出目录 | 能看到推理或训练相关产物文件 |

## 6. 验收标准

1. 核心页面可访问。
2. 核心接口可返回有效数据。
3. 推理任务链路能够完成基础演示。
4. 训练任务具备完整的入口、记录与版本组织逻辑。
5. 系统文档能够完整说明架构、模块、接口、数据和限制条件。

## 7. 当前可验证范围

### 7.1 已具备验证条件

1. 页面访问与页面结构展示。
2. 接口可达性验证。
3. 任务记录与目录结构验证。
4. 版本信息展示验证。

### 7.2 受环境影响的验证项

1. 长时训练任务完整执行。
2. 大规模数据集下的持续运行验证。
3. 多轮训练指标对比验证。

## 8. 验收结论建议写法

论文或交付说明中可写为：

“系统已完成页面、接口、任务管理、结果展示和版本归档等核心能力验证，能够支撑课题的系统演示和软件工程说明；受当前设备环境影响，完整训练过程的长时运行验证不作为本轮验收的唯一依据。”
