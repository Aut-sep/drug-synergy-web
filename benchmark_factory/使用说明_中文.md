# benchmark_factory 中文说明

这个文件夹是我专门新建的一套“统一数据集加工工厂”，目的只有一个：

把你以后准备好的“固定格式原始数据集”，自动转换成四个模型都能读取的格式，并且统一五折划分，方便公平对比。

它和你之前做交集的脚本是分开的，不会混在一起。

## 这个目录里每个文件是干什么的

### 1. `adapter_core.py`

这是最核心的文件，相当于“总加工逻辑”。

它主要负责 4 件事：

- 读取你给定格式的数据文件
- 检查数据有没有缺项，比如药物名、细胞名对不上
- 生成统一的五折划分
- 分别导出成四个模型原始代码能识别的文件结构

你可以把它理解成：

“底层工具库，真正干活的地方都在这里。”

### 2. `build_benchmark_dataset.py`

这是主入口脚本，也就是你以后真正运行的脚本。

它会调用 `adapter_core.py`，完成整套处理流程。

你以后最常用的其实就是它。

简单说：

- `adapter_core.py` 是发动机
- `build_benchmark_dataset.py` 是点火开关

### 3. `INPUT_SCHEMA.md`

这是输入数据格式说明。

它告诉你：

- 你要准备哪些文件
- 每个文件必须有哪些列
- 哪些列是可选的

如果你以后换别的数据集，只要按这个格式准备，就可以直接复用这套脚本。

### 4. `README.md`

这是整个目录的英文总说明。

内容包括：

- 这个项目是做什么的
- 导出的目录结构长什么样
- 怎么运行脚本
- 为什么四个模型可以统一比较

### 5. `summarize_fold_metrics.py`

这是“结果汇总脚本”。

等你以后把四个模型都跑完后，通常会得到每一折的指标。

这个脚本用来把五折结果做汇总，比如：

- AUC 平均值
- PR_AUC 平均值
- ACC 平均值
- Kappa 平均值

也就是统一四个模型最后的对比表。

### 6. `example_bundle`

这是我放进去的一个最小示例数据包。

它不是你的真实实验数据，只是为了测试脚本能不能正常工作。

你可以把它理解成：

“样板输入模板”

### 7. `exports`

这是脚本运行以后输出结果的地方。

目前里面有一个 `smoke_test`，是我做本地测试生成的样例输出。

里面已经按四个模型分别导出了对应目录。

## 这套脚本想解决什么问题

你之前担心的核心问题是：

1. 之前做的交集质量不可信
2. 不可能以后只跑一套交集数据
3. 四个模型原始数据接口不统一
4. 以后换数据集时不想每次重写处理代码

所以这套新方案的思路是：

不是先做一次“固定交集”然后长期硬用，
而是定义一套“统一输入规范”，以后任何数据集只要整理成这个规范，就能自动导出到四个模型。

## 你以后需要准备什么

你以后只需要准备一个输入数据文件夹，里面放这些文件：

### 1. `samples.csv`

这是主样本表，也就是组合样本本身。

至少要有这些列：

- `drug_a_name`
- `drug_b_name`
- `cell_line`
- `synergy_score`

可选列：

- `sample_id`
- `label`
- `ri_row`
- `ri_col`

### 2. `drugs.csv`

这是药物信息表。

至少要有：

- `drug_name`
- `smiles`

如果你还给了更多数值列，这些会被当作 `MTLSynergy` 的药物特征。

### 3. `cells_dualsyn.csv`

给 `DualSyn` 和 `MFSynDCP` 用的细胞特征。

### 4. `cells_mtl.csv`

给 `MTLSynergy` 用的细胞特征。

### 5. `cells_mvc_exp.csv`

给 `MVCASyn` 用的表达谱特征。

### 6. `cells_mvc_cn.csv`

给 `MVCASyn` 用的拷贝数特征。

## 它是怎么统一四个模型的

四个模型原始代码读取格式不一样，所以不能直接共用同一个 CSV。

但可以统一成“同一批样本 + 同一套五折 + 同一套标签规则”。

这套脚本做的就是这件事。

### 统一的部分

- 同一批 `(drugA, drugB, cell)` 样本
- 同一个标签定义
- 同一套五折划分
- 同一个对比协议

### 不统一但自动适配的部分

- `DualSyn` 需要自己的 `smiles.csv` 和 fold 文件
- `MFSynDCP` 需要自己的 `labels.csv` 和 fold 文件
- `MVCASyn` 需要 `exp.csv`、`cn.csv`、`oneil_drug_two_smiles.csv`
- `MTLSynergy` 需要 `drugs.csv`、`drug_features.csv`、`cell_lines.csv`、`cell_line_features.csv`、`oneil_summary_idx.csv`

这些差异由脚本自动处理。

## 五折是怎么统一的

我现在默认统一的是：

### `transductive_fold`

这是四个模型共享的一套五折划分，后面做公平对比时建议都用它。

另外我还额外生成了：

### `cell_fold`

给以后做 leave-cell-out 用。

### `drug_a_fold` 和 `drug_b_fold`

给以后做 leave-drug-out 用。

所以这套设计不是只够你现在用一次，而是给你后续扩展实验留了接口。
